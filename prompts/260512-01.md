# AllSpeak training curriculum — draft for review

> **Working document.** Goal: a self-contained curriculum that lets AI agents and human coders write idiomatic AllSpeak.
>
> Background: original idioms outline in `260429-00.md`, additions in `260429-01.md`, review and merge notes in `260512-00.md`.
>
> Drafted sections are now in `learn/`. This file remains the roadmap (TOCs), the record of decisions, and the place where outstanding work items live.

---

## 1. Proposed shape

Two tiers under a single `learn/` root.

```
learn/
  README.md
  reference/    # canonical language reference
  idioms/       # patterns and worked examples
```

**Reference** answers *what is this thing in AllSpeak?* — symbols, variables, collections, control flow, modules. Stable, canonical, encyclopedic. Re-read when memory fades.

**Idioms** answers *how do I do X the AllSpeak way?* — short pattern catalogs with worked examples, anti-patterns, and pointers back into reference. Read once per pattern, then occasionally for refresh.

Each file targets 200–600 words for an idiom; reference files can run longer (500–1500) where the concept warrants. One concept per file. Anti-patterns made explicit. Cross-references at the bottom of every file.

Format: Markdown. Headings + fenced code blocks. Language-neutral examples in English — language-specific syntax belongs in the language packs, not the curriculum.

## 6. Decisions

Outcomes from the original open questions:

1. **Location.** `learn/` at repo root.
2. **Audience tone.** Curriculum assumes some prior programming experience. First-time programmers are served by `/codex/`; no parallel ground-floor track needed.
3. **Language of examples.** English only for now. Decision on translating to FR/IT/DE deferred until the English version is complete.
4. **Cross-references.** Will be handled by the future reader (see Work items).
5. **Numbering.** Numeric prefixes retained — drafting is happening one file at a time by AI, so the renumber-on-insert cost is low.
6. **Anti-patterns.** Kept. They reinforce the message.
7. **Code-example verification.** Examples to be embedded in simple frameworks runnable directly from the reader (see Work items).
