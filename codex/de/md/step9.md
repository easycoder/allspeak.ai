Im vorherigen Schritt haben wir ein einzelnes Rechteck gezeichnet, dessen Höhe einer Sinuskurve folgte. Stellt man mehrere davon zusammen und lässt sie jeweils ein wenig nach dem vorherigen starten, entsteht eine Welle (im Stadion auch „La Ola" genannt). Der Code dafür ist etwas komplex, und ich verlange nicht, dass Sie ihn vollständig verstehen, aber ich möchte Ihnen zeigen, wie er die Arrays verwendet, die ich ein paar Schritte zuvor eingeführt habe. Wie zuvor ist es am besten, wenn Sie den Code in den Editor kopieren und dann ausführen, damit Sie sehen, wie er aussieht.

Teile des Codes ähneln dem vorherigen Schritt, aber der wesentliche Unterschied ist, dass es 10 Rechtecke sind, nicht eines. Diese werden im Behälter nebeneinander positioniert; auch hier ist das CSS schwer zu erklären, aber jeder Teil hat seinen Zweck. Die Rechtecke müssen absolute Positionierung verwenden, was bedeutet, dass sie feste Positionen haben, anstatt einfach über die Seite verteilt zu werden, wie es der Browser für richtig hält. Das würde Probleme verursachen, denn wenn sich eines von ihnen bewegt, würde es die anderen mitziehen. Komponenten können nur dann absolute Positionierung haben, wenn ihr Elternelement (der Behälter) relative Positionierung besitzt; fragen Sie mich nicht warum — es ist eben so.

Nun zu diesen Arrays. Eine Variable ist nur ein Container für einen einzelnen Wert, aber manchmal möchten wir mehrere Werte als Gruppe zusammen verwalten, wie bei den 10 Rechtecken. Wir könnten einfach ~code:Rectangle1~, ~code:Rectangle2~ und so weiter erstellen, aber dann müsste auch der gesamte Code, der sie verwaltet, dupliziert werden. Besser ist es, sie als eine einzige Variable mit 10 Elementen zu haben und jedes der Reihe nach auszuwählen, um damit zu arbeiten. Die Verwendung von Arrays reduziert den benötigten Code erheblich, und weniger Code bedeutet weniger Fehler. Die Rechtecke verhalten sich alle identisch, daher ist es einfach, eine Schleife einzurichten, um sie zu erstellen, wobei Berechnungen verwendet werden, um die Positionsunterschiede zu definieren.

Wenn das Skript läuft, führt es einen Zähler namens ~code:Time~, der ständig zunimmt. Jedes Rechteck hat einen Wert ~code:Start~, der angibt, wann es seinen Beitrag zur Welle beginnen soll; es hat auch seinen eigenen privaten ~code:Angle~ und eine Variable namens ~code:Done~, die entweder wahr oder falsch ist, je nachdem, ob der Zyklus für dieses Rechteck beendet ist. Letztere nennen wir eine _boolesche_ Variable oder ein _Flag_. Für jeden Schritt von ~code:Time~ setzt das Skript alle Arrays so, dass sie auf das erste Rechteck zeigen, und prüft, ob die Startzeit bereits erreicht wurde. Wenn ja, berechnet es die neue Position und Größe des Rechtecks und bewegt es. Dann wiederholt es den Vorgang für die anderen Rechtecke. Sobald der Zyklus für ein Rechteck beendet ist, wird das ~code:Done~-Flag gesetzt, und der Test in Zeile 69 stellt dann sicher, dass für dieses Rechteck keine weiteren Aktionen ausgeführt werden. Sobald sie alle fertig sind, beginnt der gesamte Prozess von vorne. Im vorherigen Beispiel habe ich eine Schleife innerhalb einer Schleife verwendet, aber hier habe ich stattdessen ein Label und ein explizites ~code:goto~ benutzt. Viele professionelle Programmierer tun so, als wären sie von einer solchen Praxis entsetzt, aber ich sage ihnen: „Seien Sie nicht so zimperlich — wenn es klar verständlich ist und funktioniert, ist es gut."

Wie oben gesagt, ist dies ein ziemlich komplexes Beispiel, das gewählt wurde, um zu zeigen, wie man Arrays verwendet, aber auch, um zu demonstrieren, wofür ~ec~ eingesetzt werden kann. So schwierig es auch aussehen mag, zu versuchen, es in irgendeiner anderen Computersprache umzusetzen, wäre viel schwieriger, und das entsprechende Programm wäre um ein Vielfaches länger. Der Großteil der Arbeit wird hier vom Browser erledigt; alles, was wir tun, ist, Werte in ihn hineinzugeben.
