# Interaktivität #

Wenn Ihnen holprige Formulierungen oder Fehler auffallen, schreiben Sie uns gern an [info@allspeak.ai](mailto:info@allspeak.ai) — Ihre Korrekturvorschläge helfen, die Übersetzung für künftige Nutzer zu verfeinern.*

In allen Programmen, außer den einfachsten, gibt es Möglichkeiten, wie der Benutzer interagieren und den Ablauf steuern kann. Hier zeige ich Ihnen, wie das geht. Als Beispiel bauen wir ein Skript, das den Benutzer nach einer Zahl fragt und dann diese Anzahl Schaltflächen auf dem Bildschirm erzeugt. Wenn eine Schaltfläche angeklickt wird, ändert sich ihr Erscheinungsbild, um zu zeigen, dass sie nicht mehr anklickbar ist. Klicken Sie auf die Schaltfläche, um den Code zu laden:

~copy~

Das Skript beginnt damit, den Benutzer zu fragen, wie viele Schaltflächen auf dem Bildschirm platziert werden sollen. Anschließend wird ein Paar Arrays auf die von Ihnen angegebene Zahl dimensioniert. Ein Array nimmt die Schaltflächen auf; das andere ist ein Array von Flags, die gesetzt werden, wenn die entsprechende Schaltfläche angeklickt wurde. Der nächste Codeabschnitt geht dieses Array durch und löscht jeden seiner Werte (setzt ihn auf false). Dann erstellen wir den Hauptbehälter, der die Schaltflächen aufnimmt.

Nun treten wir in eine Schleife ein, die alles jedes Mal neu zeichnet, wenn wir sie durchlaufen. Zuerst wird der Behälter von seinem gesamten vorherigen Inhalt — falls vorhanden — geleert, dann werden alle Schaltflächen mit einer Schleife gezeichnet. Für jede Schaltfläche erstellen wir den Umriss — ein ~code:div~ mit einem Rand — und prüfen dann, ob die Schaltfläche angeklickt wurde. Wenn ja, wird eine einfache Nachricht in die Schaltfläche eingefügt; wenn nicht, müssen wir einen Hyperlink erzeugen, der in HTML ein ~code:a~-Element ist, und genauso heißt er auch in ~ec~. Wir geben der Schaltfläche den Inhalt ~code:Knopf N~, wobei ~code:N~ von 0 bis zur Anzahl der Schaltflächen zählt.

Der nächste Teil beschreibt, was passiert, wenn der Benutzer eine Schaltfläche anklickt/antippt. Das Schlüsselwort "on" (bei) kennzeichnet einen Ereignishandler. Ein Ereignis ist etwas, das während der Ausführung eines Programms geschieht, und es gibt verschiedene Ereignistypen; "click" (Klick) ist einer davon. Wenn ein Klick erkannt wird, führt das System den Befehl oder Block von Befehlen aus, der folgt.

Sie können für jede Bildschirmkomponente einen Ereignishandler festlegen, aber es gibt einen wesentlichen Unterschied bei einem Hyperlink-Element: Wenn der Mauszeiger darüberfährt (darüber "schwebt"), verändert sich das Symbol zu einer Hand, während das bei einem ~code:div~ oder anderen Typen nicht der Fall ist. Das gilt natürlich nur für PCs; berührungsbasierte Geräte wie Smartphones haben keinen Hover-Mechanismus, weshalb Sie deutlich machen sollten, dass etwas ein Link ist. Ein weiterer Unterschied besteht darin, dass Hyperlinks eine "Dekoration" haben, meist in Form von unterstrichenem Text. Das ist die Voreinstellung, aber Sie können sie ändern oder in Ihren CSS-Stilen vollständig abschalten.

Die Aktion, die wir beim Erkennen eines Klicks ausführen, besteht darin, das Flag ~code:Geklickt~ für diese Schaltfläche zu setzen und dann den Bildschirm neu zu zeichnen. Wir müssen also erkennen können, welche Schaltfläche angeklickt wurde, und das ist ein sehr guter Grund, sie in einem Array zu halten. Wenn Sie ein Array von Hyperlinks haben, müssen Sie "bei klick" nur einmal definieren, weil es für das gesamte Array gilt. Wenn der Klick erfolgt, wird der interne Zeiger auf das Element gesetzt, das den Klick erhalten hat; hier holen wir diesen Index und verwenden ihn, um auf dasselbe Element im Array ~code:Geklickt~ zu zeigen. Wir setzen es auf true, sodass das Element beim Neuzeichnen des Bildschirms seine "geklickt"-Form annimmt.
